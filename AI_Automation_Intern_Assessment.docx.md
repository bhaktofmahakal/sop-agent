## **Objective**

We are looking for candidates who can use AI to solve real business problems and build practical, usable systems.

This assessment is designed to evaluate:

* Problem-solving ability  
* Practical execution  
* System thinking  
* Use of AI tools in real scenarios

## **Instructions**

* Choose ANY ONE of the problem statements below  
* Build a working solution (MVP level)  
* Focus on clarity, practicality, and execution

Your solution should:

* Take inputs

It can be:

* Fully automated (preferred)

## 🔹 Problem 1: AI Resume Screening System

### Scenario:

You receive multiple resumes for a role and need to shortlist candidates efficiently.

### Task:

Build a system that:

### Inputs:

* Job Description (JD)  
* Multiple resumes

### Outputs:

* Match score (0–100)  
* Candidate ranking  
* Key strengths (2–3 points)  
* Key gaps (2–3 points)  
* Final recommendation:  
* Strong Fit / Moderate Fit / Not Fit

### Example Output (for reference):

| Candidate | Score | Strengths | Gaps | Recommendation |
| :---- | :---- | :---- | :---- | :---- |
| A | 82 | Strong SQL, Analytics | Weak Python | Strong Fit |

---

## 🔹 Problem 2: Interview Scheduling Automation

### Scenario:

Interview scheduling involves multiple back-and-forth emails.

### Task:

Build a system that:

### Inputs:

* Candidate availability  
  (Example: Tue–Thu 2–5 PM, Fri 9 AM–12 PM)  
* Interviewer availability (3–5 interviewers)

### Outputs:

* Top 3 suggested interview time slots  
* Conflict resolution (if any overlaps)  
* Final recommendation with reasoning

### Example:

* Candidate: Tue 2–5 PM  
* Interviewer A: Tue 3–6 PM  
* Interviewer B: Tue 1–4 PM

→ Suggested slot: Tue 3–4 PM

---

## 🔹 Problem 3: Hiring Funnel Dashboard

### Scenario:

Management lacks visibility into hiring performance.

### Task:

Create a dashboard (use sample/mock data if required) showing:

* Total applicants  
* Stage-wise conversion rates  
* Time to hire  
* Drop-off points in funnel

## 🔹 Problem 4 (Optional – Advanced): SOP → AI Training System

### Scenario:

Training employees using SOP documents is manual and time-consuming.

### Task (Core \- Required):

### Input:

* SOP document (text/PDF)

### Outputs:

* Structured summary (main points)  
* Step-by-step training content  
* 3–5 evaluation/quiz questions

### SOP Input:

You may:

### Bonus (Optional \- Advanced):

* Add automation workflow
